## How to Log a Report

When a beta user reports an issue, paste one entry below using the template:

```markdown
### [BUG/UX/CONTENT/PERF/SUGGEST]-NNN — Short title

| Field | Value |
|---|---|
| Priority | 🔴 Critical / 🟠 High / 🟡 Medium / 🟢 Low |
| Reporter | Name or role (e.g. "Community Elder, iOS") |
| Device / OS | e.g. iPhone 14 / iOS 17.4, Samsung A53 / Android 14 |
| Date reported | YYYY-MM-DD |
| Reproducible | Yes / No / Sometimes |
| Status | Open / In Progress / Resolved / Won't Fix |

**Description:**  
What happened vs what was expected.

**Steps to reproduce:**  
1. 
2. 
3. 

**Resolution:**  
(fill in when fixed)
```

---

## Priority Definitions

| Level | Meaning |
|---|---|
| 🔴 Critical | App crashes, data loss, auth failure — blocks all use |
| 🟠 High | Core feature broken, no workaround |
| 🟡 Medium | Feature works but behaves unexpectedly or confusingly |
| 🟢 Low | Polish, wording, minor UX preference |

---

## Issue Categories

| Code | Meaning |
|---|---|
| BUG | Software defect |
| UX | Confusing workflow or layout |
| CONTENT | Missing, wrong, or untranslated text |
| PERF | Slow, freezing, battery/memory |
| SUGGEST | Feature request or improvement idea |
