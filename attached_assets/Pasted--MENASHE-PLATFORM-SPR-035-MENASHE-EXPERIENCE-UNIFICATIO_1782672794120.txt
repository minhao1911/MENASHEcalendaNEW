# MENASHE PLATFORM

# SPR-035 — MENASHE EXPERIENCE UNIFICATION

## Engineering Sprint Instruction (ESI)

### Role

Act as:

* Principal Product Designer
* Principal UX Architect
* Senior React Architect
* Senior Mobile UX Engineer
* Three.js Experience Designer
* Design System Architect
* Performance Engineer
* Accessibility Engineer

This is a **product unification sprint**, not a feature sprint.

---

# Mission

Transform the Menashe Platform into one unified experience.

The objective is that every screen feels like it belongs to the same application.

Users should never feel they are moving between disconnected tools.

Instead they should feel they are journeying through one living platform.

This sprint focuses entirely on experience quality.

Do NOT introduce unrelated features.

Do NOT redesign backend architecture.

Do NOT modify database schemas unless required to fix an existing UX defect.

---

# Part 1 — Home Experience

## Goal

Redesign the Home experience without rebuilding the application.

Preserve existing functionality.

Reorganize presentation and hierarchy.

---

## Desired User Journey

Greeting

↓

Today's Hebrew Date

↓

Today's Learning

↓

Today's Community

↓

Enter Memorial Sanctuary

↓

Recent Activity

---

## Design Principles

Remove dashboard clutter.

Reduce the number of competing cards.

Use generous spacing.

One primary action per viewport.

Use progressive disclosure instead of showing everything immediately.

Every section should naturally lead to the next.

The Home page should feel calm, welcoming and spiritual.

---

# Part 2 — Memorial Sanctuary Experience

Use the completed Sanctuary audit as the implementation reference.

Improve the existing SanctuaryWorld.

Do NOT create a second implementation.

Enhance the current production implementation only.

---

## Required Improvements

### Truth & Trust

Remove fabricated activity notifications.

Remove fabricated statistics.

Never display fictional users.

Never display fictional memorial activity.

If real data is unavailable, display:

"No recent activity yet."

Truth is preferred over simulated engagement.

---

### Memorial Presentation

Display real memorial photos whenever available.

Fallback order:

1. Memorial photo
2. Initials avatar
3. Memorial icon

Never default directly to emoji.

---

### Typography

Raise all text below accessibility standards.

Minimum readable size:

11px

Improve spacing and hierarchy.

Reduce visual density.

---

### Mobile

All forms must remain usable while the keyboard is open.

Prevent keyboard overlap.

Respect safe areas.

Support 320–480px widths gracefully.

No clipped panels.

No overlapping layouts.

---

### Internationalization

Every visible Sanctuary string must use the translation system.

No hardcoded English.

Support:

English

Thadou Kuki

Future Hebrew additions

---

### Performance

Reduce unnecessary re-renders.

Memoize expensive panels where appropriate.

Extract inline styles into reusable CSS where practical.

Maintain smooth frame rates.

# Part 4 — Shared Design System

Consolidate visual language.

There must be only one source of truth.

---

One accent palette.

One neutral palette.

One semantic success color.

One warning color.

One error color.

Remove duplicate gold definitions.

---

## Typography

One body scale.

One caption scale.

Consistent line heights.

Consistent font weights.

---

## Components

One button system.

One card system.

One glassmorphism style.

One loading skeleton.

One badge system.

One input system.

One modal system.

One spacing scale.

One elevation system.

---

## Design Tokens

Move repeated values into shared design tokens.

Avoid hardcoded visual values wherever possible.

# Part 5 — Truth & Trust

This is the highest priority of the sprint.

The Menashe Platform represents a real community.

Never fabricate community activity.

Never fabricate memorial statistics.

Never fabricate user names.

Never fabricate engagement.

If information is unavailable:

Show an honest empty state.

Users trust honesty more than artificial activity.
