------------------------------------------------------------
MISSION
------------------------------------------------------------

Create the official Menashe Experience Language (MEL).

MEL defines HOW the application feels.

It is NOT a Design System.

It is NOT Components.

It is NOT Business Logic.

It is the collection of experience principles that every future
screen must follow.

The goal is to ensure that Home, Calendar, Sanctuary, Torah,
Community, AI and Settings all feel like one unified product.

------------------------------------------------------------
NON-NEGOTIABLE RULES
------------------------------------------------------------

DO NOT

❌ Modify Web Application

❌ Modify Mobile Components

❌ Modify Mobile Shell

❌ Modify Business Logic

❌ Modify Shared Core

❌ Build New Screens

❌ Redesign Existing Home

This sprint creates standards only.

CHIEF ARCHITECT FINAL PRINCIPLE

MMDL defines WHAT things look like.

Components define WHAT things are.

MEL defines HOW people feel.

Every future feature must inherit MEL.

No new screen should ever invent its own experience.

Menashe Calendar should feel calm,
timeless,
warm,
respectful,
and unmistakably Menashe.

This document becomes the permanent experience constitution of the mobile application.
